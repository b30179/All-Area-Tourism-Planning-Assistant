# 安全策略

## 支持版本

| 版本 | 支持状态 |
|------|---------|
| 1.0.x | ✅ 活跃维护 |
| < 1.0 | ❌ 不再支持 |

## 报告流程

**请勿公开披露安全漏洞。** 通过以下方式私密报告：

1. 发送邮件至项目维护者（见 README）
2. 在 GitHub 上使用 [私密漏洞报告](https://github.com/b30179/All-Area-Tourism-Planning-Assistant/security/advisories/new)

我们承诺在 48 小时内确认收到，并在 2 周内提供修复方案。

## 安全实践

### 🔑 API Key 保护

- `.env` 文件**永远不会**提交到仓库（已在 `.gitignore`）
- 所有示例代码使用 `.env.example` 占位符
- 支持 Web 侧边栏输入 Key（不落盘）

### 🕷️ 爬虫安全

- crawl4ai 依赖必须 ≥ 0.9.0（低版本存在 CVSS 9.8 漏洞）
- 知识库数据库 (`*.db`) 不会入库
- 爬虫请求不会携带用户身份信息

### 📦 依赖管理

- `requirements.txt` 仅包含最小依赖
- 爬虫依赖 (`crawl4ai`) 为可选安装
- 建议在虚拟环境中运行

## 已知限制

- 本应用为教学/演示项目，不建议直接用于生产环境
- 地图 API 调用需要用户自行申请 Key（非项目提供）
- LLM 输出未经人工审核，行程规划仅供参考
